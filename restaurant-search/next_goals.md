# restaurant-search 개선 목표

## 목표

현재 슬롯 기반 배치 검색 스크립트를 **범용 tabelog 검색 도구**로 개선한다.

## 요구사항

| # | 기능 | 설명 |
|---|------|------|
| 1 | 지역 검색 | 지역명 또는 세부 지역명으로 검색 (일본 전역 지원) |
| 2 | 카테고리 검색 | 카테고리로 가게 특정 (예: ラーメン, 焼肉 등) |
| 3 | 최소 점수 필터 | tabelog 점수 N점 이상만 조회 |
| 4 | 최소 리뷰수 필터 | 리뷰 N개 이상만 조회 |
| 5 | 페이지네이션 제어 | 0~N 페이지까지 조회 범위 지정 |
| 6 | 검색 제한 | 총 가져온 가게 수가 N개를 넘으면 검색 종료 |

## 현재 제약사항

- tabelog는 공식 API가 없음 — HTML 스크래핑에 의존
- CSS 셀렉터 변경 시 파싱이 깨질 수 있음
- 요청 간 딜레이(현재 3초) 필요 — 과도한 요청 시 차단 가능성
- 한국어 이름 미제공 — `name` = `nameJa`로 동일

## 미결 사항 (논의 필요)

### 1. 지역 검색 방식

tabelog URL 구조가 두 가지 존재:

| 방식 | URL 예시 | 장점 | 단점 |
|------|----------|------|------|
| 프리텍스트 (`sw=`) | `/osaka/rstLst/?sw=道頓堀` | 구현 간단, 자유도 높음 | 키워드 미매칭 시 누락 |
| 지역코드 | `/osaka/A2701/A270101/rstLst/` | 정확한 지역 필터링 | 코드 매핑 테이블 필요 |

→ 둘 다 지원? 프리텍스트만?

### 2. 카테고리 검색 방식

tabelog는 카테고리별 URL 코드 제공 (예: `RC0111` = 라멘):

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 주요 카테고리만 매핑 | 자주 쓰는 20~30개만 하드코딩 | 간단, 유지보수 쉬움 | 커버리지 제한 |
| 전체 카테고리 크롤링 | tabelog 카테고리 페이지에서 전체 수집 | 완전한 커버리지 | 초기 구현 복잡 |
| 키워드 검색만 | `sw=` 파라미터에 카테고리명 포함 | 매핑 불필요 | 정확도 낮음 |
