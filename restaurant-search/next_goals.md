# restaurant-search 개선 목표

## 목표

현재 슬롯 기반 배치 검색 스크립트를 **범용 tabelog 검색 도구**로 개선한다.

## 요구사항

| # | 기능 | 설명 |
|---|------|------|
| 1 | 지역 검색 | 지역명 또는 세부 지역명으로 검색 (일본 전역 지원) |
| 2 | 카테고리 검색 | 카테고리로 가게 특정 (예: ラーメン, 焼肉 등) |
| 3 | 최소 점수 필터 | tabelog 점수 N점 이상만 조회 |
| 4 | 최소 리뷰수 필터 | 리뷰 N개 이상만 조회 |
| 5 | 페이지네이션 제어 | 0~N 페이지까지 조회 범위 지정 |
| 6 | 검색 제한 | 총 가져온 가게 수가 N개를 넘으면 검색 종료 |

## 현재 제약사항

- tabelog는 공식 API가 없음 — HTML 스크래핑에 의존
- CSS 셀렉터 변경 시 파싱이 깨질 수 있음
- 요청 간 딜레이(현재 3초) 필요 — 과도한 요청 시 차단 가능성
- 한국어 이름 미제공 — `name` = `nameJa`로 동일

## 결정사항

### 1. 지역 검색 방식 → 프리텍스트 우선

- **v2**: 프리텍스트 검색(`sw=`) 사용
- **후순위**: 지역코드(`/A2701/...`) 기반 정밀 검색 추가
- **설계 고려**: 지역코드 확장 시 검색 인터페이스 변경 없이 내부 URL 빌더만 교체 가능하도록 설계

### 2. 카테고리 검색 방식 → 주요 카테고리 매핑

- **v2**: 주요 카테고리 20~30개 코드 매핑 (예: `ramen` → `RC0111`)
- **후순위**: 전체 카테고리 코드 크롤링/자동 수집
- **설계 고려**: 카테고리 매핑을 별도 데이터 파일로 분리하여 확장 용이하게

### 3. CLI 인터페이스 → argparse 기반

- 새 스크립트(`search.py`)에서 `argparse`로 옵션 처리
- 예: `python search.py --area "道頓堀" --category ramen --score 3.0 --reviews 200 --pages 5 --limit 30`

### 4. 기존 슬롯 모드 → 별도 파일 유지

- 기존 `main.py` + `slots.py`는 그대로 유지 (슬롯 배치 전용)
- 새 자유 검색은 `search.py`로 분리
- `tabelog.py`는 공통 모듈로 양쪽에서 사용

### 5. 출력 형식 → CSV + JSON + 터미널 테이블

- `--format csv` (기본값): 파일 저장
- `--format json`: JSON 파일 저장
- `--format table`: 터미널에 표로 출력
