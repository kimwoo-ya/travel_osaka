# restaurant-search v2 — 엣지케이스 분석

## 핵심 발견사항 (즉시 대응 필요)

| ID | 문제 | 빈도 | 영향도 | 설명 |
|----|------|------|--------|------|
| EC-DATA-001 | 리뷰 수 쉼표 오파싱 | 높 | 높 | `re.findall(r"\d+", "1,234件")` → `["1","234"]` → 1로 파싱. 인기 가게 탈락 |
| EC-NET-005 | HTML 구조 변경 무감지 | 중 | 높 | bare `except: continue`가 파싱 에러를 삼킴. 200개 전부 실패해도 "0건"으로 표시 |
| EC-NET-001 | HTTP 403/429 차단 | 중 | 높 | 403과 500 미구분, 재시도 불가한 차단에 18초 낭비 |
| EC-DATA-002 | 평점 '-' → 0.0 변환 | 중 | 중 | 미평가와 극저평점 구분 불가. `--score 0` 시 동작 모호 |
| EC-DATA-005 | 이름 기반 중복 제거 | 중 | 중 | 체인점 다른 지점 미감지, 동명이점 오탈락 |

## 1. CLI 입력 경계값

### --score
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 음수 (-1.0) | 낮 | 중 | argparse에서 0.0~5.0 범위 검증, 에러 메시지 |
| 5 초과 (6.0) | 낮 | 낮 | 범위 검증 (tabelog 최대 5.0) |
| 문자열 ("abc") | 낮 | 중 | argparse type=float로 자동 거부 |
| 미입력 | 높 | 낮 | 기본값 0 (필터 없음) |

### --reviews
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 음수 (-10) | 낮 | 중 | 0 이상 범위 검증 |
| 0 | 중 | 낮 | 허용 (필터 없음과 동일) |
| 매우 큰 값 (999999) | 낮 | 낮 | 허용, 결과 0건 예상 |

### --pages
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 0 | 중 | 중 | 1 이상 강제 또는 에러 |
| 음수 | 낮 | 중 | 1 이상 범위 검증 |
| 매우 큰 값 (100) | 낮 | 높 | 상한선 설정 (예: 20). rate limit 위험 |

### --limit
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 0 | 낮 | 중 | 1 이상 강제 또는 에러 |
| 음수 | 낮 | 중 | 1 이상 범위 검증 |
| 미입력 | 높 | 낮 | 기본값 무제한 또는 적절한 상한 |

### --area
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 빈 문자열 | 낮 | 중 | 필수 파라미터로 설정, 미입력 시 에러 |
| 한국어 입력 ("도톤보리") | 중 | 중 | 일본어 검색이 정확. 안내 메시지 고려 |
| 특수문자 포함 | 낮 | 낮 | URL 인코딩으로 처리 |

### --category
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 매핑에 없는 카테고리 | 중 | 중 | 에러 + 사용 가능 카테고리 목록 표시 |
| 일본어 직접 입력 (ラーメン) | 중 | 낮 | 프리텍스트 검색으로 폴백 |
| 미입력 | 높 | 낮 | 선택 사항, 미입력 시 전체 카테고리 |

### --format
| 입력 | 빈도 | 영향도 | 권장 대응 |
|------|------|--------|----------|
| 지원하지 않는 포맷 | 낮 | 중 | argparse choices로 자동 거부 |
| 미입력 | 높 | 낮 | 기본값 csv |

## 2. 네트워크/스크래핑 에러

| ID | 시나리오 | 빈도 | 영향도 | 권장 대응 |
|----|----------|------|--------|----------|
| EC-NET-001 | HTTP 403 (차단) | 중 | 높 | 재시도 중단, "접근 차단됨" 메시지 |
| EC-NET-002 | HTTP 429 (rate limit) | 중 | 높 | `Retry-After` 헤더 준수, 적응형 딜레이 |
| EC-NET-003 | HTTP 500 (서버 에러) | 낮 | 중 | 기존 재시도 로직 유지 |
| EC-NET-004 | 타임아웃 | 중 | 중 | 현재 15초 타임아웃 적절. 재시도 유지 |
| EC-NET-005 | HTML 구조 변경 | 낮 | 높 | 파싱 실패율 추적. 50%+ 시 경고 출력 |
| EC-NET-006 | 연결 실패 (DNS/네트워크) | 낮 | 높 | 첫 요청 실패 시 즉시 중단 + 네트워크 확인 안내 |
| EC-NET-007 | CAPTCHA 페이지 반환 | 낮 | 높 | HTML에 검색 결과 없고 CAPTCHA 감지 시 중단 |

## 3. 데이터 품질 문제

| ID | 문제 | 빈도 | 영향도 | 권장 대응 |
|----|------|------|--------|----------|
| EC-DATA-001 | 리뷰 수 쉼표 (`1,234` → `1`) | 높 | 높 | 쉼표 제거 후 파싱: `text.replace(",", "")` |
| EC-DATA-002 | 평점 '-' → 0.0 | 중 | 중 | `None`으로 처리, 필터에서 제외 |
| EC-DATA-003 | 가격 정보 없음 | 높 | 낮 | 빈 문자열 허용, 출력에서 "미확인" 표시 |
| EC-DATA-004 | 가게명 특수문자/이모지 | 낮 | 낮 | UTF-8 유지, CSV는 utf-8-sig |
| EC-DATA-005 | 이름 기반 중복 제거 오류 | 중 | 중 | URL 기반으로 전환 |
| EC-DATA-006 | 카테고리 없음 (빈 문자열) | 중 | 낮 | "미분류"로 표시 |
| EC-DATA-007 | 복합 카테고리 (`焼肉 / ホルモン`) | 중 | 낮 | `/` split 후 첫 번째만 사용 (현재 동작 유지) |
| EC-DATA-008 | 가격 순서 반전 (디너/런치) | 낮 | 중 | index 하드코딩 유지, 문서화 |

## 4. 출력 관련 에러

| ID | 시나리오 | 빈도 | 영향도 | 권장 대응 |
|----|----------|------|--------|----------|
| EC-OUT-001 | 결과 0건 (CSV) | 중 | 중 | 헤더만 있는 빈 파일 + 안내 메시지 |
| EC-OUT-002 | 결과 0건 (JSON) | 중 | 중 | 빈 배열 `[]` + 안내 메시지 |
| EC-OUT-003 | 결과 0건 (table) | 중 | 중 | "결과 없음" 메시지 출력 |
| EC-OUT-004 | CSV 데이터에 쉼표/따옴표 | 중 | 중 | `csv.writer`가 자동 이스케이프 (현재 동작 OK) |
| EC-OUT-005 | 터미널 테이블 긴 텍스트 | 중 | 낮 | 최대 폭 제한 + 말줄임 |
| EC-OUT-006 | JSON 인코딩 이슈 | 낮 | 낮 | `ensure_ascii=False` 사용 |
| EC-OUT-007 | 파일 쓰기 권한 없음 | 낮 | 높 | try/except + 명확한 에러 메시지 |

## 5. Success Criteria

| ID | 기준 | 검증 방법 |
|----|------|----------|
| SC-001 | 사용자가 1개 명령어로 지역+카테고리 검색을 완료할 수 있다 | `--area` + `--category` 조합 실행 |
| SC-002 | 검색 결과가 2분 이내에 반환된다 (10페이지 기준) | 시간 측정 |
| SC-003 | 점수/리뷰 필터가 정확히 동작한다 | 결과의 모든 가게가 기준 충족 확인 |
| SC-004 | 3가지 출력 형식(CSV/JSON/table)이 모두 동작한다 | 각 포맷으로 동일 검색 실행 |
| SC-005 | 기존 슬롯 배치 모드가 영향받지 않는다 | `python main.py O2` 기존 동작 확인 |
| SC-006 | 카테고리 매핑 20개 이상이 정확히 동작한다 | 각 카테고리로 검색 실행 |
| SC-007 | 결과 0건 시 사용자에게 명확한 안내가 제공된다 | 존재하지 않는 지역으로 검색 |
| SC-008 | `--limit` 옵션이 검색을 정확히 제한한다 | limit=5 설정 후 결과 수 확인 |
| SC-009 | 잘못된 입력 시 유용한 에러 메시지가 표시된다 | 범위 외 값, 미지원 카테고리 입력 |
| SC-010 | 네트워크 에러 시 사용자에게 원인이 안내된다 | 오프라인 상태에서 실행 |
| SC-011 | 리뷰 1,000건 이상 가게가 정확히 파싱된다 | 쉼표 포함 리뷰 수 검증 |
| SC-012 | 중복 가게가 결과에 포함되지 않는다 | 동일 URL 중복 확인 |
| SC-013 | CSV/JSON 출력 파일이 다른 도구에서 정상 열린다 | Excel, jq 등으로 확인 |
| SC-014 | `--help` 옵션이 모든 파라미터를 설명한다 | help 출력 확인 |
| SC-015 | 페이지 제한이 총 요청 수를 적절히 통제한다 | pages=1 vs pages=10 요청 수 비교 |

## 6. Assumptions

| ID | 가정 |
|----|------|
| AS-001 | 사용자는 안정적인 인터넷 연결 환경에서 실행한다 |
| AS-002 | tabelog의 HTML 구조가 단기간(수개월) 내 급격히 변경되지 않는다 |
| AS-003 | 3초 요청 간격이 tabelog의 rate limit을 초과하지 않는다 |
| AS-004 | 검색어는 일본어로 입력하는 것이 가장 정확하다 |
| AS-005 | 카테고리 코드는 tabelog 실제 확인 후 확정한다 (현재 추정치) |
| AS-006 | 기존 main.py/slots.py는 수정하지 않는다 |
| AS-007 | Python 3.10+ 환경에서 실행한다 |
| AS-008 | 출력 파일은 스크립트 실행 디렉토리에 저장한다 |
| AS-009 | 동시 실행(멀티 프로세스)은 v2 범위 밖이다 |
| AS-010 | tabelog 로그인 없이 공개 데이터만 수집한다 |
